Что мы вообще сделали, по сути
Раньше ваше приложение проверяло пароль только по своей локальной базе (PostgreSQL) — как обычный сайт. Теперь мы добавили вторую, приоритетную проверку: перед тем как смотреть в свою базу, приложение сначала спрашивает у Active Directory: "этот пароль верный для этого человека?"
Представьте это как охранника на входе с двумя списками:

Список от службы безопасности компании (AD) — проверяется первым.
Свой запасной список (ваша база данных) — используется, только если охранник не смог связаться со службой безопасности.

Как это работает по шагам (когда кто-то заходит на сайт)

Человек вводит логин и пароль на странице входа.
Ваш сервер берёт эти данные и стучится в AD (по защищённому протоколу LDAPS) с вопросом: "Существует ли пользователь с таким логином, и совпадает ли пароль?"
Если AD говорит "да" — сервер дополнительно спрашивает: "А в какой он группе?" Если человек состоит в группе HR_Managers — он получает роль Chief Manager (admin). Если нет — обычную роль Manager.
Если это первый вход этого человека — сервер сам создаёт запись о нём в вашей локальной базе (чтобы остальная часть приложения, которая работает через ID пользователя, продолжала работать как обычно).
Если AD недоступен или ошибся — сервер не отказывает сразу, а идёт во второй, запасной список — вашу локальную базу с паролями, как это было раньше.

Это и есть та самая идея, которую озвучивал руководитель: пароли теперь не вы выдаёте вручную — ими управляет сама компания через AD, а ваша база — просто подстраховка.
Как добавить нового пользователя (сейчас, в тестовом AD)
Три команды в терминале (в контейнере samba-ad):
powershell# Создать пользователя
docker exec -it samba-ad samba-tool user create petrova P@ssw0rd456 --given-name=Мария --surname=Петрова

# Если человек должен быть Chief Manager — добавить в группу
docker exec -it samba-ad samba-tool group addmembers HR_Managers petrova
Если не добавлять в группу HR_Managers — человек всё равно сможет войти, но получит обычную роль Manager (не admin), это заложено в логике по умолчанию.
